# 09 并行执行与失败降级

## 1. 目标

一次审稿可能启用多个 Agent。系统应该并行执行，提高速度；同时任何单个 Agent 失败，都不应该让整次审稿失败。

## 2. 执行策略

推荐使用 `asyncio.gather`。

```python
results = await asyncio.gather(*tasks, return_exceptions=True)
```

对于 exception，要转成 `AgentResult(status="failed")`。

## 3. 并发数量限制

不要无限并发。配置：

```python
class ReviewSettings(BaseModel):
    max_concurrent_agents: int = 4
```

使用 semaphore：

```python
sem = asyncio.Semaphore(settings.max_concurrent_agents)

async def run_with_sem(agent):
    async with sem:
        return await runner.run_reviewer(...)
```

## 4. 执行顺序

虽然并行执行，但任务构造前先按 priority 排序：

```python
selected_agents = sorted(
    selected_agents,
    key=lambda a: (-a.priority, a.agent_id),
)
```

结果返回后也按这个顺序排序，方便报告稳定。

## 5. Agent 级上下文过滤

运行每个 Agent 前：

```python
context_for_agent = filter_context_for_agent(
    retrieved_context,
    agent,
    max_chunks=settings.max_context_chunks_per_agent,
)
```

不要把所有上下文都塞给每个 Agent。

## 6. Agent 失败分类

统一错误类型：

```text
llm_timeout
llm_api_error
json_parse_error
schema_validation_error
empty_output
unknown_error
```

AgentResult 中记录：

```json
{
  "agent_id": "risk_reviewer",
  "agent_name": "舆情风险审查员",
  "status": "failed",
  "findings": [],
  "error_message": "json_parse_error: ..."
}
```

## 7. 关键 Agent 失败处理

某些 Agent 失败需要在报告中显著提示。

配置：

```python
critical_agent_ids = ["risk_reviewer", "privacy_reviewer"]
```

如果关键 Agent 失败：

```text
审稿继续。
final_report.warnings 加入：关键 Agent 失败，本次报告可能漏掉相关风险。
报告顶部显示。
```

## 8. 非关键 Agent 失败处理

例如 format_reviewer 失败：

```text
记录到 Agent 明细。
不影响最终报告生成。
不提高总风险等级。
```

## 9. 输出为空处理

Agent 成功但 findings 为空：

```text
status=success
summary 显示“未发现该维度明显问题”
不视为错误
```

## 10. 超时处理

每个 Agent 使用自己的 timeout_seconds。

如果底层 SDK 没有内置超时，外层包一层：

```python
await asyncio.wait_for(task, timeout=agent.model.timeout_seconds)
```

超时返回：

```python
AgentResult(
    agent_id=agent.agent_id,
    agent_name=agent.name,
    status="timeout",
    findings=[],
    error_message="Agent execution timed out",
)
```

## 11. 失败是否重试

重试发生在 LLMClient 内部。

策略：

```text
网络/API 错误：按 max_retries 重试。
JSON 解析错误：只做一次 repair。
Schema 错误：只做一次 repair。
业务校验错误：不重试，直接 invalid_output。
```

## 12. 失败日志

每个失败 Agent 在 debug 文件中写：

```text
agent_id
stage
error_type
error_message
raw_output
```

路径：

```text
output/{run_id}/debug/{agent_id}_error.json
```

## 13. 验收测试

必须写：

```text
test_parallel_executor_limits_concurrency
test_one_agent_failure_does_not_stop_others
test_timeout_returns_timeout_agent_result
test_critical_agent_failure_adds_warning
test_empty_findings_is_success
```
