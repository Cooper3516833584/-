# 10 仲裁、去重、评分与报告分层

## 1. 目标

多个 Agent 会产生大量意见，其中会有重复、弱意见、过度审查、格式问题和真正重要的问题。

仲裁层要完成：

```text
去重
合并
评分
分层
排序
输出最终报告结构
```

第一版不要求使用 LLM 做仲裁，先用规则仲裁，稳定可控。

## 2. Finding 收集

从所有成功 AgentResult 中收集 findings：

```python
def collect_findings(agent_results: list[AgentResult]) -> list[Finding]:
    findings = []
    for result in agent_results:
        if result.status == "success":
            findings.extend(result.findings)
    return findings
```

失败 Agent 不产生 findings。

## 3. 基础清洗

清洗规则：

```text
删除 original_quote 为空的 finding。
删除 problem 为空的 finding。
删除 suggestion 为空的 finding。
把 original_quote strip。
把 risk_level 不合法的 finding 降为 info 或丢弃。
```

建议不轻易丢弃，除非严重缺字段。

## 4. 去重规则

### 4.1 精确重复

同一个 original_quote + issue_type 相同，合并。

### 4.2 近似重复

第一版可以用简单相似度：

```python
from difflib import SequenceMatcher

def text_similarity(a: str, b: str) -> float:
    return SequenceMatcher(None, a, b).ratio()
```

规则：

```text
original_quote 相似度 > 0.85 且 issue_type 相同或 tags 有交集，则认为重复。
```

### 4.3 同段重复

如果 segment_id 相同，且 issue_type 相近，也可以合并。

## 5. 合并策略

合并后的 Finding 应保留：

```text
最高 risk_level
最高 confidence
所有来源 Agent
所有 evidence 去重
最长或最具体的 problem
最具体的 suggestion
tags 合并去重
```

可以扩展 Finding：

```python
class MergedFinding(Finding):
    source_agent_ids: list[str]
    source_agent_names: list[str]
    merged_from: list[str]
    score: float
    final_bucket: str
```

第一版也可以在 Finding.metadata 中保存这些字段。

## 6. 风险等级分值

```python
RISK_LEVEL_SCORE = {
    "high": 90,
    "medium": 60,
    "low": 30,
    "info": 10,
}

CONFIDENCE_MULTIPLIER = {
    "high": 1.0,
    "medium": 0.75,
    "low": 0.45,
}
```

## 7. 加权因素

```text
有 article evidence：+5
有 rules evidence：+10
有 cases evidence：+8
有 risky_phrases evidence：+5
被多个 Agent 命中：每多一个 Agent +6，最多 +18
issue_type 包含 privacy/authorization：+10
issue_type 包含 fact_error：+6
issue_type 包含 format：-10
confidence=low：最终不允许进入“必须修改”，除非 risk_level=high 且有 rules/cases 依据
```

## 8. 评分函数

```python
def score_finding(f: Finding, source_agent_count: int) -> float:
    base = RISK_LEVEL_SCORE[f.risk_level]
    score = base * CONFIDENCE_MULTIPLIER[f.confidence]

    source_types = {e.source_type for e in f.evidence}

    if "article" in source_types:
        score += 5
    if "rules" in source_types:
        score += 10
    if "cases" in source_types:
        score += 8
    if "risky_phrases" in source_types:
        score += 5

    score += min(max(source_agent_count - 1, 0) * 6, 18)

    tags = set(f.tags)
    if "privacy" in tags or "authorization" in tags:
        score += 10
    if "format" in tags:
        score -= 10

    return max(score, 0)
```

## 9. 分层规则

```python
def assign_bucket(score: float, finding: Finding) -> str:
    if finding.confidence == "low" and finding.risk_level != "high":
        return "reference"

    if score >= 85:
        return "must_fix"
    if score >= 60:
        return "should_fix"
    if score >= 35:
        return "optional"
    return "reference"
```

中文显示：

```text
must_fix    必须修改
should_fix  建议修改
optional    可选优化
reference   仅供参考
```

## 10. 特殊强制规则

无论分数如何，以下情况至少进入“建议修改”：

```text
risk_level=high 且 confidence=medium/high
隐私授权问题且 risk_level=medium/high
事实错误且 confidence=high
```

以下情况不能进入“必须修改”：

```text
只有 style_guides 或 examples 依据的风格建议
confidence=low 且没有 rules/cases 依据
issue_type 是 format 且 risk_level 不是 high
```

## 11. 排序规则

每个 bucket 内排序：

```text
score 从高到低
risk_level 从 high 到 info
confidence 从 high 到 low
source_agent_count 从多到少
原文出现顺序从前到后
```

## 12. 最终报告结构

```python
class FinalReport(BaseModel):
    summary: dict
    selector: dict
    buckets: dict[str, list[dict]]
    agent_execution: list[dict]
    knowledge_used: list[dict]
    warnings: list[str]

```

summary 包含：

```text
total_findings
must_fix_count
should_fix_count
optional_count
reference_count
overall_risk_level
critical_warnings
```

## 13. overall_risk_level 规则

```text
如果 must_fix_count >= 1：high
否则如果 should_fix_count >= 2：medium
否则如果 should_fix_count == 1 或 optional_count >= 3：low
否则：info
```
