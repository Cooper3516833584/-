# 00 实施总览

## 1. 项目定位

本项目是一个 Python 单次智能审稿工具。

它的完整执行链路是：

```text
读取稿件
  ↓
读取根目录下的 agents/cases/rules/style_guides/risky_phrases/examples 等 txt 文件
  ↓
预处理稿件：识别标题、正文、段落、元信息
  ↓
Selector Agent 判断稿件类型、风险重点、需要启用哪些审稿 Agent
  ↓
知识检索器根据稿件内容与 Selector 结果检索相关规矩、翻车案例、风格规范
  ↓
并行运行被选中的审稿 Agent
  ↓
校验 Agent 输出是否符合统一 JSON Schema
  ↓
合并重复问题，做风险评分和仲裁分层
  ↓
生成 Markdown / JSON 审稿报告
  ↓
保存到 output/
```

## 2. 不做什么

明确不实现：

```text
用户账号系统
权限管理
团队组织管理
稿件多轮流转
主编审批流
消息通知
数据库后台管理
发布到公众号/网站
在线多人协作编辑器
```

可选实现：

```text
FastAPI 单接口
简易网页上传页面
本地向量索引缓存
报告导出为 docx/html
```

但第一版必须先完成 CLI 版。

## 3. 推荐技术栈

第一版核心依赖：

```text
Python >= 3.11

typer                # CLI 命令
rich                 # CLI 进度条和漂亮输出
python-dotenv        # 读取 .env

orjson               # JSON 读写
tenacity             # LLM 调用重试
openai               # LLM API 客户端，可替换为其它供应商
```

可选依赖：

```text
numpy                # 向量缓存
scikit-learn         # TF-IDF 或相似度检索
rank-bm25            # BM25 检索
fastapi              # 可选 Web API
uvicorn              # 可选 Web API 运行
python-docx          # 可选 docx 导出
```

## 4. 最小可用版本必须具备的能力

MVP 验收要求：

```text
1. 能从 input/article.md 或命令行参数读取稿件。
2. 能加载 agents/*.txt，至少包含 selector、fact_checker、risk_reviewer、privacy_reviewer、format_reviewer。
3. 能加载 cases/rules/style_guides/risky_phrases/examples 下的 txt 文件。
4. Selector Agent 能输出 selected_agent_ids。
5. 系统能对 Selector 结果做合法性校验，防止选择不存在或禁用的 Agent。
6. 被选中的 Agent 能并行或顺序运行。
7. 每个 Agent 的输出必须转成统一 Finding 列表。
8. 非法输出要自动修复一次或记录失败。
9. 仲裁器能把问题分为：必须修改、建议修改、可选优化、仅供参考。
10. 输出 output/{run_id}/report.md 和 output/{run_id}/report.json。
```

## 6. 一次审稿的运行命令

第一版 CLI 目标：

```bash
review-tool review input/article.md --output output
```

支持指定配置根目录：

```bash
review-tool review input/article.md --project-root . --output output
```

支持只看 Selector 选择了哪些 Agent：

```bash
review-tool select-agents input/article.md
```

支持检查配置文件是否合法：

```bash
review-tool validate-config
```

支持重建本地知识索引：

```bash
review-tool index
```
