# 01 项目边界与最终能力

## 1. 核心用户故事

编辑把一篇稿件交给工具，工具自动完成：

```text
1. 判断这是什么稿件。
2. 判断应该启用哪些审稿 Agent。
3. 根据稿件内容检索相关校规、翻车案例、风格规范。
4. 让多个 Agent 从不同角度审稿。
5. 合并重复意见。
6. 去掉明显无依据的弱意见。
7. 输出编辑可直接使用的审稿报告。
```

## 2. 单次审稿定义

“单次审稿工具”的含义：

```text
每次运行只处理一篇稿件。
每次运行都独立生成一个 run_id。
每次运行结果写入 output/{run_id}/。
系统不维护用户状态，不维护稿件生命周期。
系统不需要登录，不需要权限。
系统不处理多轮审批，只输出审稿结果。
```

## 3. 输入

### 3.1 最小输入

```text
稿件标题
稿件正文
```

### 3.2 推荐输入格式

支持 Markdown 文件：

```markdown
---
title: 新生节活动回顾：在热闹中重新认识校园
author: 校媒编辑部
column: 校园生活
article_type: unknown
images:
  - cover.jpg
  - activity_01.jpg
---

# 新生节活动回顾：在热闹中重新认识校园

正文第一段……

正文第二段……
```

如果没有 front matter，则使用以下策略：

```text
第一个一级标题作为 title。
剩余内容作为 body。
article_type 设为 unknown，由 Selector Agent 判断。
column 为空。
images 为空。
```

## 4. 输出

### 4.1 JSON 输出

保存到：

```text
output/{run_id}/report.json
```

包含：

```text
run_id
article_summary
selector_result
agent_results
merged_findings
arbitration_result
final_report
errors
metadata
```

### 4.2 Markdown 输出

保存到：

```text
output/{run_id}/report.md
```

面向编辑阅读，必须包含：

```text
审稿总览
Selector Agent 判断
启用的 Agent 列表
必须修改
建议修改
可选优化
仅供参考
Agent 执行明细
知识库引用摘要
失败与警告
```

## 5. 系统必须做到的控制点

### 5.1 Agent 不能随便输出自由文本

所有审稿 Agent 都必须输出 JSON。

即便 prompt 以后再细化，代码层也必须要求：

```json
{
  "agent_id": "risk_reviewer",
  "status": "success",
  "findings": []
}
```

### 5.2 Selector Agent 不能拥有最终控制权

Selector Agent 负责推荐启用哪些 Agent，但代码必须校验：

```text
Agent 是否存在
Agent 是否 enabled
是否是 selector 本身
是否超过最大 Agent 数量
是否漏掉基础必跑 Agent
是否选择了与稿件完全无关的 Agent
```

### 5.3 Selector 失败时必须兜底

兜底策略：

```text
启用基础 Agent：
- fact_checker
- risk_reviewer
- audience_reviewer
- format_reviewer
- privacy_reviewer

如果有图片：再启用 copyright_reviewer
如果疑似采访：再启用 interview_ethics_reviewer
如果标题含强情绪词：再启用 title_reviewer
```

### 5.4 知识库检索不能阻塞审稿

如果 cases/rules 等目录为空，审稿仍要运行。报告中提示：

```text
未检索到本地知识材料，本次审稿主要基于稿件文本和 Agent 通用审查规则。
```

## 6. 最终能力清单

### 6.1 Agent 管理

```text
通过 agents/*.txt 新增 Agent
通过 enabled 控制 Agent 是否启用
通过 applies_to 控制 Agent 适用范围
通过 priority 控制执行优先级
通过 max_findings 控制每个 Agent 最大输出条数
```

### 6.2 自动选择 Agent

```text
Selector Agent 读取稿件 + Agent catalog
输出 selected_agent_ids + 理由
代码层校验并补充基础 Agent
```

### 6.3 本地 txt 知识库

```text
cases/ 翻车案例
rules/ 规矩制度
style_guides/ 风格规范
risky_phrases/ 高风险表达
examples/ 正例样稿
```

### 6.4 审稿输出

```text
每条问题必须包含原文摘录
每条问题必须包含风险等级
每条问题必须包含修改建议
每条问题应尽量包含知识库依据
每条问题标明来源 Agent
```

### 6.5 报告分层

```text
必须修改：高风险、高置信度、多个 Agent 命中、或涉及隐私授权
建议修改：中风险或有明确优化价值
可选优化：表达、可读性、格式
仅供参考：低置信度、低概率误读、证据不足但值得注意
```
