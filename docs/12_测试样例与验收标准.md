```text
Selector 失败有兜底
知识库能检索
Agent 输出能校验
重复问题能合并
```


## 6. 端到端测试

```python
def test_review_engine_e2e(tmp_path):
    project_root = copy_fixture_project(tmp_path)
    engine = ReviewEngine(project_root=project_root, settings=ReviewSettings(llm_provider="mock"))
    result = await engine.review_file(project_root / "input" / "sample_article.md", project_root / "output")

    assert result.run_id
    assert result.final_report
    assert (project_root / "output" / result.run_id / "report.md").exists()
    assert (project_root / "output" / result.run_id / "report.json").exists()
```

## 7. 人工验收样例

准备 5 类样例稿件：

```text
1. 普通安全稿：应输出少量格式/表达建议，不应乱判高危。
2. 隐私风险稿：包含学生姓名、照片、班级，应触发 privacy_reviewer。
3. 采访稿：包含问答和引语，应触发 interview_ethics_reviewer。
4. 活动报道：包含图片和赞助信息，应触发 copyright_reviewer/title_reviewer。
5. 标题党稿：标题夸张但正文中性，应触发 title_reviewer 和 risk_reviewer。
```

## 8. 验收标准

MVP 完成标准：

```text
review-tool validate-config 成功。
review-tool show-agents 能列出 Agent。
review-tool select-agents 能输出 selected_agent_ids。
review-tool review 能生成 report.md 和 report.json。
Selector 失败时仍能审稿。
至少 5 个 Agent 可以被加载和运行。
知识库为空时不崩溃。
知识库存在时报告能展示引用。
测试通过率 90% 以上。
```

## 9. 质量验收标准

审稿报告应满足：

```text
每条问题有原文摘录。
每条问题有修改建议。
高风险问题有清晰理由。
低置信度问题不会被放到必须修改。
重复问题不会反复出现。
Agent 失败会明确提示。
```
